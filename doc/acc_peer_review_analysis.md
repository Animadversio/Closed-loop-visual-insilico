# Accentuation Peer-Review Analysis

## Core Concept

This analysis tests the **accentuation failure hypothesis** using real primate neural data.

In a closed-loop neural experiment, a model is used to generate stimuli via gradient accentuation
(maximize model-predicted response). The question is: does the true neural response also increase?

We define two roles:
- **Accentuator**: the model that *generated* the stimuli (gradient accentuation target)
- **Reviewer**: the model whose predictions we compare against the actual neural responses

The key experiment is a 2×2 matrix of (accentuator × reviewer) combinations, examining whether
neural responses track the reviewer's predictions on accentuator-generated images.

### Models compared
- `resnet50` — standard ImageNet-trained ResNet-50 (Ridge regression readout)
- `resnet50_robust` — adversarially robust ResNet-50 (L∞ ε=8/255)

The robust model has a different internal feature structure — lower-frequency, more "brain-like"
features — which is the key contrast for the accentuation experiment.

### Stimulus types
- **NSD images** (`shared{id}_nsd{id}.png`): 200 natural images from the Natural Scenes Dataset
- **Accentuated images**: generated by gradient accentuation using each model as accentuator.
  Filename format: `{model}_{readout}_unit_{u}_img_{i}_level_{v}_score_{s}.png`

## Data Tables

### `df_allstim_w_neural_rsp`
All stimuli from the accentuation session. Built by:
1. Parsing all stimulus filenames via `parse_accentuated_filenames_to_df`
2. Adding per-unit neural response columns: `neural_resp_unit_{u}`

Columns include: `filepath`, `model_name`, `unit_id`, `img_id`, `level`, `score`,
`neural_resp_unit_0`, `neural_resp_unit_1`, ..., `neural_resp_unit_63`

- `model_name` is NaN for NSD natural images
- `unit_id` is the unit the stimulus was generated to drive

### `df_merged`  (NSD images only)
Merge of `df_natimg_w_neural_rsp` (NSD subset of above) with `encoding_stim_info_w_pred_resp`:
- Left key: `filepath`, Right key: `stimulus_name`
- Adds columns: `pred_resp_{model}_unit_{u}`, `is_nsd`, `is_test`, `is_train`, etc.

### `accentuated_merged_df`  (accentuated images only)
Merge of `accentuated_stim_info_w_pred_resp` (model predictions) with `df_allstim_w_neural_rsp`
(neural responses), joined on `(basename/filepath, img_id, unit_id, model_name)`.
Sanity checked: `level` and `score` columns must match between tables.

## Figures

### 1. NSD Validation (`fig_nsd_stacked`)
- y = neural response, x = predicted response (each reviewer model)
- One panel per reviewer, stacked vertically, shared axes
- Tests basic encoding model quality on natural images

### 2. Accentuated 2×2 (`fig_acc_2x2`)
- 2×2 grid: rows = accentuator model, cols = reviewer model
- y = neural response, x = reviewer's predicted response
- Points colored by `img_id` (each seed image is a different color)
- Stats annotated: Pearson r, R², N

**Key interpretation:**
- Diagonal (accentuator == reviewer): model's own predictions on its own-generated images
- Off-diagonal: cross-model generalization
- If accentuation works correctly (model=brain): r should be high in all cells
- If accentuation fails (model ≠ brain): neural responses don't track the accentuator's predictions
